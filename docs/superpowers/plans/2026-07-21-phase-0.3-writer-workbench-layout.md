# Phase 0.3 Writer Workbench Layout Implementation Plan

> **For Codex:** REQUIRED SKILL: Use `superpowers:subagent-driven-development` to execute this plan task by task in the current session. Apply `superpowers:test-driven-development` to every behavior change and `superpowers:verification-before-completion` before any completion claim.

**Goal:** Turn the verified Writing Buddy Code-OSS baseline into a runnable writer-oriented three-column workbench prototype with a native Markdown editor, three switchable demo chapters, a static scene assistant, live frozen-rule word counts, and restart-safe state.

**Architecture:** Add one dependency-free built-in extension at `extensions/writing-buddy`. The extension contributes a primary-sidebar Tree View, a secondary-sidebar Webview View, status-bar entries, writer-oriented configuration defaults, and commands that open ordinary Markdown `TextDocument`s in the native Monaco editor. It bootstraps a checked-in demo novel into the extension global-storage directory without overwriting existing files, opens that directory as the workspace on first run, and stores only UI pointers such as the last chapter; Code-OSS remains responsible for document content, save, hot exit, undo/redo, external-file conflict detection, editor state, and workspace restoration. No `src/**` patch is planned; add one build-list entry so the normal built-in-extension pipeline compiles the extension.

**Tech Stack:** Code-OSS built-in extension API, TypeScript ES2024/CommonJS, native Monaco Markdown editor, TreeDataProvider, WebviewViewProvider, StatusBarItem, Mocha TDD tests, existing gulp extension build, Electron source runtime.

**Frozen Contract:** `docs/plans/phase-0.3-writer-workbench-layout-contract.md`

export function countWords(text: string): number {
